# Spear Phishing as a Violation of the Domain Knowledge Graph

Reproducibility package for the paper:

> **Spear Phishing as a Violation of the Domain Knowledge Graph:
> Cascade Dynamics and Leak-Aware Evaluation**
> Kamil Warpechowski, Bogdan Księżopolski.

Modern email filters catch mass phishing (~98%), but the residual ~2% — spear phishing and
Business Email Compromise (BEC) — slips through *precisely because* it respects the victim's
**domain knowledge**. We model that knowledge as a **multiplex graph** and detect targeted attacks
via two invariants — **cross-layer inconsistency** and **temporal cascade dynamics** — measure
**organizational susceptibility**, and define detection effectiveness under a **leak-aware** protocol.

## Repository layout

```
.
├── paper/        LaTeX sources (Elsevier CAS single-column) + figures + bibliography
│   ├── main.tex          entry point (\input{en/...})
│   ├── en/               manuscript sections (abstract body + 6 sections)
│   ├── figures/          12 figures used by the paper (PDF)
│   ├── bibliography.bib  94 cited references
│   ├── highlights.txt    research highlights (≤85 chars each)
│   └── cas-*.cls/.sty/.bst  Elsevier CAS class + bib style
├── code/         experiment code (Python)
│   ├── graph/            domain-knowledge-graph layer construction (multiplex + OSINT + temporal)
│   ├── data/             synthetic-population generators + the generated twins/ (150 personas)
│   ├── cascadebench/     leak-aware cascade testbed, detectors, statistics
│   ├── experiments/      runnable experiments (one file per result)
│   ├── analysis/         plotting / figure generation (English labels)
│   └── results/          computed experiment outputs (CSVs) — figures reproduce from these
└── references/
    ├── library/  51 analyzed papers on the topic (47 with full PDF + reading notes):
    │             spear phishing, BEC, lateral, OSINT, graph/GNN fraud & anomaly detection
    └── cited/    one folder per formally cited reference (94): bib metadata + DOI/arXiv link,
                  analyzed note where available, PDF where obtainable
```

> **What is bundled.** All **generated/computed** artifacts are versioned: the synthetic population
> `code/data/twins/` (150 personas) and every experiment output in `code/results/` (CSVs, multiplex
> layers, OSINT overlays) — so the tables and figures are reproducible offline. The large **public**
> input datasets (Enron maildir, SNAP graphs, the Elliptic set, ≈0.9 GB) and temporary caches are
> **not** bundled; see `code/README.md` for download pointers.

## Quick start

### Build the paper
```bash
cd paper
pdflatex main
bibtex   main
pdflatex main
pdflatex main
```
(Requires a TeX Live with the Elsevier CAS class; the `cas-*` files are bundled. Output: `paper/main.pdf`.)

### Reproduce experiments
```bash
python -m venv venv && . venv/bin/activate      # (Windows: venv\Scripts\activate)
pip install -r requirements.txt
# example: the organizational-susceptibility measurement
python code/experiments/exp_susceptibility.py
```
Each script under `code/experiments/` writes a CSV to `code/results/` and is named after the
result it produces. See `code/README.md` for the full map of experiments → tables/figures.

## Data and ethics

All experiments use a **synthetic** population of related personas (no real personal data, no GDPR
concerns) and **public** communication graphs (Enron, SNAP) whose *structure* is real but whose
attack *labels* are controlled injections. We validate the **mechanism and topology**, not the
detection of real-world incidents.

## Archiving (Zenodo)

This repository is published as a citable archive. To rebuild the upload package:

```powershell
powershell -ExecutionPolicy Bypass -File make_zenodo_archive.ps1
```

The script produces `../graph-phishing-spear-detection-zenodo-v1.0.0.zip` containing `paper/` and
`code/` only. The `references/` folder (third-party PDFs) is **deliberately excluded** — those files
are other authors' copyright and are kept locally for private research use, not redistributed.
